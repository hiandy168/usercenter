伪静态设置

CMS借助YII框架强大的路由管理组件，可实现url规则的多样化，可将原本超长的且不美观的url变化优雅，默认已经内定几条相关规则。

如正常的地址：http://www.soft.cn/index.php?r=page/show&name=about  

重写为 

http://www.soft.cn/page/show/about

只需要以下2个步骤即可开启，但是需要服务器支持。

第一步：修改系统配置文件中的路由组件 urlManager 部分改成以下内容

view source
print?
01'urlManager'=>array(
02            'urlFormat'=>'path',
03            'showScriptName'=>false,
04            //'urlSuffix'=>'.html',
05            'rules'=>array(
06                'list/catalog_<catalog:\d+>/*'=>'list/index',
07                'post/<id:\d+>/*'=>'post/show',
08                'post/<id:\d+>_<title:\w+>/*'=>'post/show',
09                'post/catalog/<catalog:[\w-_]+>/*'=>'post/index',
10                'page/show/<name:\w+>/*'=>'page/show',
11                '<controller:\w+>/<action:\w+>'=>'<controller>/<action>',
12            ),
13        ),


实际上就是将urlFormat 和 showScriptName 前台的注释 // 去掉，若需要在 url后台带上 .html 如：http://www.soft.cn/page/show/about.html，则需要将 urlSuffix前面的注释 // 去掉

第二步：配置WEB服务，让支WEB服务的软件支持URL重写，下面介绍常见的WEB软件支持URL重写的配置方法

IIS6：

在IIS6下要支持伪静态必须安装第三方的组件 ISAPI_Rewrite，装完此软件后在网站根目录新建一文件 http.ini 内容为：


view source
print?
1[ISAPI_Rewrite]
2CacheClockRate 3600
3RepeatLimit 32
4RewriteRule (.*\.(js|ico|gif|jpg|jpeg|png|swf|css|php|rar|zip|xml|ani|cur|mp3|mp4|flv|exe))$ /$1 [L,U]
5RewriteRule (?!/static)(.*)$ /index.php/$1 [L]


即可支持URL的重写，至于ISAPI_Rewrite软件的下载和安装，请自行baidu：http://www.baidu.com/s?wd=ISAPI_Rewrite

IIS7.X

IIS7/IIS7.5可使用微软官方的组件 http://www.iis.net/downloads/microsoft/url-rewrite 直接安装后即可支持，配置文件名称是 web.config，内容为


view source
print?
01<?xml version="1.0" encoding="UTF-8"?>
02<configuration>
03    <system.webServer>
04        <rewrite>
05            <rules>
06                <rule name="Imported Rule 1">
07                    <match url="." ignoreCase="false" />
08                    <conditions logicalGrouping="MatchAll">
09                        <add input="{REQUEST_FILENAME}" matchType="IsFile" ignoreCase="false" negate="true" />
10                        <add input="{REQUEST_FILENAME}" matchType="IsDirectory" ignoreCase="false" negate="true" />
11                    </conditions>
12                    <action type="Rewrite" url="index.php" />
13                </rule>
14            </rules>
15        </rewrite>
16    </system.webServer>
17</configuration>


若你的网站web.config已经有其它配置，不能直接覆盖，可以将 <rewrite></rewirte> 之间的内容复制到你WEB的配置文件web.config中即可



NGINX:

nginx支持伪静态只需要在nginx server配置段加入伪静态规则即可，以 lnmp 环境为例：

lnmp是将伪静态的配置段单独放到文件中再加载到lnmp中的



所以需要编辑网站配置中的伪静态文件，加入以下内容


view source
print?
1location / {
2    if (!-e $request_filename){
3        rewrite ^/(.*) /index.php last;
4    }
5}


APACHE:

apache默认的伪静态文件名称是 .htaccess ，内容为：

view source
print?
01<IfModule mod_rewrite.c> 
02  Options +FollowSymLinks 
03  IndexIgnore */* 
04  RewriteEngine on 
05     
06    # if a directory or a file exists, use it directly 
07  RewriteCond %{REQUEST_FILENAME} !-f 
08  RewriteCond %{REQUEST_FILENAME} !-d 
09   
10  # otherwise forward it to index.php 
11  RewriteRule . index.php 
12</IfModule>


同样也是将配置文件 .htaccess 放在网站根目录下，打开网站测试是否可以正常浏览，若提示404错误，说明apache伪静态组件是未开启的，需要手动去开启，步骤以下：

1.打开apache配置文件，一般在apache安 装目录 conf/httpd.conf ，找到组件配置项   #LoadModule rewrite_module modules/mod_rewrite.so  将前面的 # 注释去掉




2.找到 <Directory 配置段，将 AllowOverride 后面的 None 改成 All
